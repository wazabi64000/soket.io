Introduction au projet de chat en temps réel

Objectif du guide

Ce guide a pour objectif de vous apprendre à développer une application de messagerie en temps réel, avec envoi et réception instantanée de messages, gestion d’utilisateurs connectés, et stockage des données en base. Nous allons utiliser les technologies modernes suivantes :

  • React : pour l’interface utilisateur
  • Express (ESM) : pour le serveur Node.js
  • Socket.io : pour la communication temps réel
  • MongoDB : pour le stockage des messages

Présentation de Socket.io

Socket.io est une bibliothèque JavaScript qui permet de gérer des connexions WebSocket. Elle offre une abstraction qui simplifie le développement de fonctionnalités temps réel (comme un chat, des notifications, ou des mises à jour en direct).

Elle gère automatiquement la reconnexion, le fallback sur d'autres protocoles si WebSocket n'est pas disponible, et propose une API simple pour échanger des événements entre client et serveur.

Exemple de fonctionnement

Une fois connecté, un client peut émettre un événement vers le serveur :

socket.emit('message', 'Bonjour tout le monde');

Et le serveur peut écouter cet événement :

io.on('connection', (socket) => {
  socket.on('message', (msg) => {
    console.log('Message reçu :', msg);
  });
});

Installation de Socket.io

Sur le serveur (Node.js)

Installez la bibliothèque via npm :

npm install socket.io

Sur le client (React)

Du côté client, installez le package suivant dans le dossier React :

npm install socket.io-client

Pré-requis pour suivre ce guide

  • Node.js installé (version 16+ recommandée)
  • MongoDB local ou via Atlas
  • Connaissances de base en JavaScript et React
  • Terminal Linux ou macOS, ou WSL/Git Bash sous Windows
